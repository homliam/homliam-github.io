hexo+github+域名搭建全过程

发表于 2019-03-26 | 阅读次数：

只需要花费一个小时，便可以在互联网的汪洋里建立属于自己的一方天地，何乐不为？—-记录一次hexo轻量级博客搭建过程。

一、搭建hexo

1、安装 git

直接在 git官网下载最新版本的 git，右键安装，安装位置可以自己选，一路 NEXT。

2、安装最新版本的 nodejs

同上nodejs官网下载最新版本安装，安装位置一路默认，也可以自己选。 如果没有安装在C盘，必须要自己配置一下环境，否则nodejs会自己配置。安装在非系统盘可以节省C盘空间，作为128G SSD 的我装在了非系统盘，配置过程参考教程，非常详细，一次成功。第一次安装过程中就是

三、配置自己的域名登陆博客

1、购买域名和解析

  • 既然是用自己的域名登陆，第一件事情肯定是要购买一个域名，我的域名是在腾讯云购买的，并且已经备案完成homliam.cn建议前往阿里云购买，目前 .top 和 .club 等域名比较便宜，只要几块钱。并且不需要备案。
  • 购买过程就不概述了，无非就是选一个名字，然后扫码支付即可。

  • 域名买好之后就是解析域名，推荐腾讯云的解析平台 https://www.dnspod.cn/ 微信注册登陆即可，并且提前在购买域名的网站修改域名DNS为；

  • 登陆后点击域名解析，在域名记录里添加下面两条内容：

    记录值中的usrname改成自己的用户名即可，注意最有有一个. 不过书忘记了也没有关系，网站会自动补全。

2、github设置和新建CNAME文件

  • 在github项目设置中增加自己的域名，保存。

  • 在博客根目录文件夹下,例如我的D:\blog\source，新建名为CNAME的记事本文件，去掉后缀。在里面输入你的域名，例如我的：homliam.cn 即可，保存并关闭。

这时候就可以用自己的域名进行登陆了，例如我的：homliam.cn

四、结语

  • 配置过程顺利的话应该都不要一个小时的时间，但是后面的主题配置才是最浪费时间的，只有一个漂亮的、独一无二的主题才是一个个人博客的精髓。在没有一点点的前端基础上，我前后几天花费了大概有4个小时才到了现在这样勉强能看的地步。不过作为一个新手，搭建成功的成就感还是相当棒的。
  • 主题我用的是目前github上排名第一的NEXT,详细配置我也是参考了几位大牛的经验来完成，还有很多不满意的地方，以后再陆续修改吧。不过暂时是不太想折腾了，由于本人专业并不是程序员，作为一名精装修行业的从业人员，平时工作也是很繁重的，本着不进则退的原则，有时间还得累积自己的工作经验和学习新的工作技能，实在没有太多精力投入到页面美化中去了，只能一点一点的来优化。
  • 今天写下这篇搭建过程，主要是给自己做一个笔记，再回忆一遍搭建思路。如果有幸能被某位网友看到这篇文章，并且借鉴我的经验成功搭建hexo，那便是这篇文章的一点价值吧！
